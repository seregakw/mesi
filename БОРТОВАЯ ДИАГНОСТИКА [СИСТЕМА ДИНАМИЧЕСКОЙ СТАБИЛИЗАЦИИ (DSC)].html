Описание проверки, выполняемой системой бортовой диагностики (OBD)

•  Проверка, выполняемая системой бортовой диагностики, служит для определения целостности и работоспособности системы динамической стабилизации. Система бортовой диагностики выводит результаты проверки по специальному запросу.
•  Кроме того, система бортовой диагностики:
―  Выполняет быструю проверку системы динамической стабилизации, которая обычно проводится в начале каждой процедуры диагностики.
―  Обеспечивает сверку данных после ремонтных работ, что позволяет убедиться в отсутствии появления новых неисправностей в ходе ремонта.
•  Проверка, выполняемая системой бортовой диагностики, состоит из трех частей:
―  Считывание/удаление результатов диагностики, мониторинг и запись PID, режим активной подачи команд.

Считывание/удаление результатов диагностики

•  Данная функция позволяет считывать и удалять коды неисправности из памяти блока DSC HU/CM.


мониторинг и запись PID/данных

•  Данная функция обеспечивает доступ к некоторым параметрам, входным сигналам, расчетным величинами и информации о статусе системы.


Режимы активной подачи команд

•  Данная функция позволяет управлять работой устройств с помощью M-MDS.


Процедура считывания диагностических кодов неисправности

1.  Подключите диагностическую систему M-MDS к диагностическому разъему DLC-2.

am3zzw00001632

2.  После идентификации автомобиля выберите следующие пункты меню в окне начального этапа диагностической системы M-MDS.

•  При использовании IDS (портативного ПК)
1.  Выберите “Toolbox”.
2.  Выберите “Self Test”.
3.  Выберите “Modules”.
4.  Выберите “ABS”.
•  При использовании PDS (карманного ПК)
1.  Выберите “Module Tests”.
2.  Выберите “ABS”.
3.  Выберите “Self Test”.

3.  Проверьте диагностический код неисправности в соответствии с указаниями, появляющимися на экране.

•  В случае вывода на дисплей какого-либо диагностического кода неисправности выполните процедуру поиска и устранения неисправностей, соответствующую этому диагностическому коду неисправности.

4.  После устранения неисправности удалите все диагностические коды неисправности из памяти блока управления DSC. (См. Процедура удаления диагностического кода неисправности.)


Процедура удаления диагностического кода неисправности

1.  Подключите диагностическую систему M-MDS к диагностическому разъему DLC-2.

am3zzw00001632

2.  После идентификации автомобиля выберите следующие пункты меню в окне начального этапа диагностической системы M-MDS.

•  При использовании IDS (портативного ПК)
1.  Выберите “Toolbox”.
2.  Выберите “Self Test”.
3.  Выберите “Modules”.
4.  Выберите “ABS”.
•  При использовании PDS (карманного ПК)
1.  Выберите “Module Tests”.
2.  Выберите “ABS”.
3.  Выберите “Self Test”.

3.  Проверьте диагностический код неисправности в соответствии с указаниями, появляющимися на экране.

4.  Находясь в экране кодов неисправностей, нажмите кнопку удаления (clear), чтобы удалить коды неисправностей.

5.  Убедитесь, что коды неисправностей больше не отображаются.


Процедура мониторинга и записи PID/данных

1.  Подключите диагностическую систему M-MDS к диагностическому разъему DLC-2.

am3zzw00001632

2.  После идентификации автомобиля выберите следующие пункты меню в окне начального этапа диагностической системы M-MDS.

•  При использовании IDS (портативного ПК)
1.  Выберите “Toolbox”.
2.  Выберите “DataLogger”.
3.  Выберите “Modules”.
4.  Выберите “ABS”.
•  При использовании PDS (карманного ПК)
1.  Выберите “Module Tests”.
2.  Выберите “ABS”.
3.  Выберите “DataLogger”.

3.  Выберите соответствующие параметры (PID) из таблицы параметров.

4.  Выполните проверку значений параметров в соответствии с указаниями, появляющимися на экране.

Примечание
•  Функция наблюдения значений параметров применяется для отслеживания расчетных значений параметров. Поэтому, если значение наблюдаемого параметра исполнительного устройства не укладывается в допустимые пределы, требуется проверка наблюдаемых параметров соответствующих исполнительных устройств. То есть, поскольку система не воспринимает неисправность исполнительного устройства, как отклонение наблюдаемого параметра от нормы, нужно проверять каждое исполнительное устройство в режиме активной подачи команд.

Процедура режима активной подачи команд

Примечание
•  Использование режима активной подачи команд при работающем двигателе может привести к записи кода неисправности U2064 в памяти блока управления приборной панелью. Вот почему следует использовать режим активной подачи команд только при неработающем двигателе. Если вы случайно включили режим активной подачи команд при работающем двигателе, проверьте память блока управления приборной панелью на предмет кодов неисправности. При наличии удалите их.

1.  Подключите диагностическую систему M-MDS к диагностическому разъему DLC-2.

am3zzw00001632

2.  После идентификации автомобиля выберите следующие пункты меню в окне начального этапа диагностической системы M-MDS.

•  При использовании IDS (портативного ПК)
1.  Выберите “Toolbox”.
2.  Выберите “DataLogger”.
3.  Выберите “Modules”.
4.  Выберите “ABS”.
•  При использовании PDS (карманного ПК)
1.  Выберите “Module Tests”.
2.  Выберите “ABS”.
3.  Выберите “DataLogger”.

3.  Выберите соответствующий режим активной подачи команд из таблицы PID.

4.  Войдя в режим активной подачи команд, проверьте работу каждого компонента.

•  Отсутствие звука включения реле, электродвигателя или электромагнитного клапана во время проверки в режиме активной подачи команд может говорить об обрыве или коротком замыкании цепи проводного жгута, реле, электродвигателя или электромагнитного клапана, а также об их заедании или иной неисправности.
